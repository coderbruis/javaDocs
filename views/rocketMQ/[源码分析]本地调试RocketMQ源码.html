CoderBruis 2021-08-03 21:59:26 RocketMQ

RocketMQ版本：4.9.0

# 1. 下载好源码并到入IDEA中并构建

详细请看

RocketMQ源码搭建

# 2. 启动rocketmq-console

在网上下载好rocketmq-console之后，通过命令：

mvn clean install
1

在target包下启动rocketmq-console

java -jar rocketmq-console-ng-1.0.0.jar
1

启动之后页面如下：

此时还没有启动源码，所以没有集群信息。

# 3. 启动NameServer

先找到NamesrvController所在目录（namesrv模块下）

启动NameServer之前，需要配置ROCKETMQ_HOME环境变量，用于读取ROCKETMQ目录下的conf等配置文件。所以在IDEA中，需要先配置Environment variables变量：

ROCKETMQ_HOME=D:\MyGitHubProject\rocketmq-sourcecode-analysis\rocketmq-rocketmq-all-4.9.0
1

如下图：

并且需要再rocketmq源码目录下新建一个conf包用于读取配置文件，如下图：

broker-a.properties内容如下：

#所属集群名字
brokerClusterName=rocketmq-cluster
#broker名字，名字可重复,为了管理,每个master起一个名字,他的slave同他,eg:Amaster叫broker-a,他的slave也叫broker-a
brokerName=broker-a
#0 表示 Master，>0 表示 Slave
brokerId=0
#nameServer地址，分号分割
namesrvAddr=127.0.0.1:9876
#在发送消息时，自动创建服务器不存在的topic，默认创建的队列数
defaultTopicQueueNums=4
#是否允许 Broker 自动创建Topic，建议线下开启，线上关闭
autoCreateTopicEnable=true
#是否允许 Broker 自动创建订阅组，建议线下开启，线上关闭
autoCreateSubscriptionGroup=true
#Broker 对外服务的监听端口,
listenPort=10911
#删除文件时间点，默认凌晨 4点
deleteWhen=04
#文件保留时间，默认 48 小时
fileReservedTime=120
#commitLog每个文件的大小默认1G
mapedFileSizeCommitLog=1073741824
#ConsumeQueue每个文件默认存30W条，根据业务情况调整
mapedFileSizeConsumeQueue=300000
#destroyMapedFileIntervalForcibly=120000
#redeleteHangedFileInterval=120000
#检测物理文件磁盘空间
diskMaxUsedSpaceRatio=88
#存储路径  这里报错了！！
#storePathRootDir=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\broker-a
#commitLog 存储路径
#storePathCommitLog=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\broker-a\commitlog
#消费队列存储路径存储路径
#storePathConsumeQueue=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\broker-a\consumequeue
#消息索引存储路径
#storePathIndex=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\broker-a\index
#checkpoint 文件存储路径
#storeCheckpoint=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\checkpoint
#abort 文件存储路径
#abortFile=D:\Developer_SoftWare\rocketmq-all-4.9.0-bin-release\store\abort
#限制的消息大小
maxMessageSize=65536
#flushCommitLogLeastPages=4
#flushConsumeQueueLeastPages=2
#flushCommitLogThoroughInterval=10000
#flushConsumeQueueThoroughInterval=60000
#Broker 的角色
#- ASYNC_MASTER 异步复制Master
#- SYNC_MASTER 同步双写Master
#- SLAVE
brokerRole=ASYNC_MASTER
#刷盘方式
#- ASYNC_FLUSH 异步刷盘
#- SYNC_FLUSH 同步刷盘
flushDiskType=ASYNC_FLUSH
#checkTransactionMessageEnable=false
#发消息线程池数量
#sendMessageThreadPoolNums=128
#拉消息线程池数量
#pullMessageThreadPoolNums=128

# 3. 启动Broker

同样的，启动Broker前需要为其设置程序参数（Program arguments），指定broker配置文件，此处的配置文件在第二节中已经配置好了。

找到BrokerStartup启动类，选中之后对其启动配置进行修改。

命令如下：

-c D:\MyGitHubProject\rocketmq-sourcecode-analysis\rocketmq-rocketmq-all-4.9.0\conf\broker-a.properties
1

如图：

启动之后控制台显示如下内容则表示启动成功：

The broker[broker-a, 10.22.12.138:10911] boot success. serializeType=JSON and name server is 127.0.0.1:9876
1

NameServer和Broker启动之后，查看rocketmq-console可以看到Broker集群以及注册进来了。
