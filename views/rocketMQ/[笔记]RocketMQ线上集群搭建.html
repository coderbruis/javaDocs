CoderBruis 2021-08-06 22:59:26 RocketMQ

# 1. 搭建过程

# 1.1 准备服务器

对于RocketMQ集群搭建，首先得准备好至少两台线上服务器（本人使用的是2核8G），对于如何通过“按量付费使用阿里云ECS”，参考本人的文章：

阿里云ECS按量付费

最终服务器信息如下：

服务器 内存 CPU IP
CentOS 7.3 8G 2核 121.40.xxx.113
CentOS 7.3 8G 2核 121.40.xxx.145

# 1.2 为服务器准备Java

在CentOS的服务器上安装JDK环境，可以参考文章：

CentOS 7 安装 JAVA环境（JDK 1.8）

# 1.3 下载并安装RocketMQ

下载地址如下：

https://archive.apache.org/dist/rocketmq/4.5.1/rocketmq-all-4.5.1-bin-release.zip
1

在Linux环境中，直接通过如下命令即可在Linux中下载：

wget https://archive.apache.org/dist/rocketmq/4.5.1/rocketmq-all-4.5.1-bin-release.zip
1

在两台服务器的/opt/software目录下分别下载好RocketMQ之后，配置hosts，配置内容如下：

121.40.xxx.145 rocketmq-nameserver1
121.40.xxx.145 rocketmq-master1
121.40.xxx.113 rocketmq-slave2

# 1.5 修改启动配置文件

分别在两台机器上修改启动配置文件：

vim /usr/local/rocketmq/rocketmq-all-4.4.0-bin-release/bin/runbroker.sh
1

修改runbroker.sh内容，内容如下：

...
JAVA_OPT="${JAVA_OPT} -server -Xms2G -Xmx2G -Xmn1G"
...
1
2
3
vim /usr/local/rocketmq/rocketmq-all-4.4.0-bin-release/bin/runserver.sh
1

修改runserver.sh内容，内容如下：

JAVA_OPT="${JAVA_OPT} -server -Xms2G -Xmx2G -Xmn1G -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=320m"
1

就这样，配置文件的准备工作就完成了。

# 3. 安装RocketMQ-console

在/opt/software目录下运行以下命令：

git clone https://github.com/apache/rocketmq-externals
1
cd rocketmq-externals/rocketmq-console/
1
yum -y install maven
1

进入rocketmq-console同pom.xml目录，运行如下命令进行打包：

mvn clean package -Dmaven.test.skip=true
1
vim /opt/software/rocketmq-externals/rocketmq-console/src/main/resources/application.properties
1

修改内用如下：

rocketmq.config.namesrvAddr=rocketmq-nameserver1;rocketmq-nameserver2
1

然后进入打好包的target目录下，启动jar

java -jar rocketmq-console-ng-1.0.1.jar
1

这样，就可以在rocketmq-console的网页上查看RocketMQ集群状态了。

# 4. 问题配排查

# 4.1 broker节点启动不成功？

按配置文件broker-a-s.properties来配置时，发现broker的nohup日志一直没结果，并且报253，去排查~/logs/rocketmqlogs/broker.log，发现报：

Try to start service thread:AllocateMappedFileService started:false lastThread:null
1

并且，启动不了broker节点，发现是由于

/usr/local/src/rocketmq/store/{commitlog,consumequeue,index}
1

这几个配置文件导致的，

因此删掉/usr/local/src/rocketmq/store和/usr/local/rocketmq/store这两个目录下的文件之后，重新新建这两个目录下的commitlog,consumequeue,index即可，

随后重启broker即可。
