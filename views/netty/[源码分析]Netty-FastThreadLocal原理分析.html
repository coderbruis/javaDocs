# 2.1 FastThreadLocal的创建

首先对于FastThreadLocal的构造方法，可以看到

    public FastThreadLocal() {
        // 原子类索引递增
        index = InternalThreadLocalMap.nextVariableIndex();
    }
1
2

可以看到通过一个原子类递增返回给index，而index在FastThreadLocal的定义如下：

    private final int index;
1

index没有被static修饰，说明每个FastThreadLocal对象都有一个自己的index成员属性。而InternalThreadLocalMap的nextVariableIndex方法和其内部的AtomicInteger原子类都是static静态修饰的，表明通过InternalThreadLocalMap静态调用nextVariableIndex能够获取唯一的一个index，确保了FastThreadLocal能独立的分布在JVM的堆中。所以，FastThreadLocal的index属性是作为FastThreadLocal的唯一身份标识。

# 2.2 FastThreadLocal的get方法实现

    public final V get() {
        // 获取InternalThreadLocalMap对象
        InternalThreadLocalMap threadLocalMap = InternalThreadLocalMap.get();
        // 通过index索引来获取map中存的对象
        Object v = threadLocalMap.indexedVariable(index);
        if (v != InternalThreadLocalMap.UNSET) {
            return (V) v;
        }

        // 初始化一个对象，暴露的方法用户可以自定义
        return initialize(threadLocalMap);
    }

调用来到InternalThreadLocalMap#get()方法，在展开之前先看下InternalThreadLocalMap的类继承结构以及成员变量。

对于InternalThreadLocalMap类，是作为FastThreadLocalThread的成员变量，而InternalThreadLocalMap中定义了一个Object[]数组对象，用于存储每个FastThreadLocal存储的对象。

public final class InternalThreadLocalMap extends UnpaddedInternalThreadLocalMap {
    // 用于对非FastThreadLocalThread线程的map
    private static final ThreadLocal<InternalThreadLocalMap> slowThreadLocalMap =
            new ThreadLocal<InternalThreadLocalMap>();
    // 为初始化时Object[]数组默认值，表示未存储对象
    public static final Object UNSET = new Object();
    // 对象数组（初始大小为32）
    Object[] indexedVariables;
}

    private static InternalThreadLocalMap fastGet(FastThreadLocalThread thread) {
        // FastThreadLocalThread本身就维护了一个ThreadLocalMap
        InternalThreadLocalMap threadLocalMap = thread.threadLocalMap();
        if (threadLocalMap == null) {
            // 给当前线程的InternalThreadLocalMap设置一个值
            thread.setThreadLocalMap(threadLocalMap = new InternalThreadLocalMap());
        }
        return threadLocalMap;
    }

接着，调用栈来到了InternalThreadLocalMap#indexedVariable

    public Object indexedVariable(int index) {
        Object[] lookup = indexedVariables;
        return index < lookup.length? lookup[index] : UNSET;

由于此时indexedVariables的object[]中并未缓存对象，所以取出的时候UNSET。则最终会调用FastThreadLocal#initialize方法初始化一个对象，并缓存到InternalThreadLocalMap中的indexedVariables数组里。

    private V initialize(InternalThreadLocalMap threadLocalMap) {
        V v = null;
        try {
            v = initialValue();
        } catch (Exception e) {
            PlatformDependent.throwException(e);
        }
        // 将新初始化出的对象缓存到indexedVariable中
        threadLocalMap.setIndexedVariable(index, v);
        addToVariablesToRemove(threadLocalMap, this);
        return v;
    }

上文分析的是走FastThreadLocalThread线程的调用栈，下面分析下走普通Thread的调用栈。

    public static InternalThreadLocalMap get() {
        Thread thread = Thread.currentThread();
        if (thread instanceof FastThreadLocalThread) {
            // 通过FastThreadLocalMap来获取
            return fastGet((FastThreadLocalThread) thread);
        } else {
            // 通过ThreadLocal来获取（由于ThreadLocal相比于FastThreadLocal来说是慢的，所以这里是slowGet）
            return slowGet();
        }
    }

    private static InternalThreadLocalMap slowGet() {
        // 通过JDK的ThreadLocal去拿InternalThreadLocalMap对象
        InternalThreadLocalMap ret = slowThreadLocalMap.get();
        if (ret == null) {
            // 如果ThreadLocal中没有InternalThreadLocalMap对象，则直接new一个出来并存到ThreadLocal中，随后返回
            ret = new InternalThreadLocalMap();
            slowThreadLocalMap.set(ret);
        }
        return ret;
    }
